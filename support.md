# Support Information
**Last updated:** 26 June 2026

## Covered Projects

This support policy currently applies to:

* TeamBridge
* MyPersonalWiki-Bot (Personal Recent Wiki)

Additional community projects may be added in the future.

---

## Getting Support

Support, bug reports, feature requests and general questions can be submitted through one of the following channels:

* GitHub Issues (preferred for bugs and feature requests)
* Discord (where a project community or support server is available)
* Other project-specific support channels, when explicitly provided

When reporting a problem, please include as much relevant information as possible, such as:

* Project name
* Software version or commit (if known)
* Error messages
* Steps to reproduce the issue
* Relevant screenshots or log excerpts

Providing detailed information helps identify and resolve issues more efficiently.

---

## Response Times

These projects are developed and maintained by volunteers.

Support is provided on a **best-effort** basis, and no guaranteed response times or service-level agreements (SLAs) are offered.

While every effort is made to review reports and respond when possible, response times may vary depending on contributor availability.

---

## Feature Requests

Suggestions for new features and improvements are welcome.

Feature requests may be evaluated based on:

* Project goals
* Technical feasibility
* Available development time
* Community benefit

Submitting a feature request does not guarantee that it will be implemented.

---

## Commercial Support

Commercial support, consulting and priority assistance are **not currently offered** for these projects.

---

## Security Issues

If you believe you have discovered a security vulnerability, please avoid publishing sensitive details publicly.

Instead, contact the project maintainer privately through Discord or another available contact method so the issue can be investigated before public disclosure.
